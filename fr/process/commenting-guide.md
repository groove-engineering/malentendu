# Donnez votre avis sur *Le Malentendu* — guide rapide (5 min, zéro compétence technique)

Bonjour,

On soumet la **méthodologie** du projet au débat ouvert, et votre avis compte vraiment.
Voici comment le laisser — c'est plus simple qu'il n'y paraît.

## 1. Avoir un compte GitHub
C'est tout ce qu'il faut — le dépôt est **public**, pas d'invitation nécessaire.
Pas de compte ? Créez-en un gratuit en 2 minutes sur **github.com** (un email, un mot de passe).

## 2. Ouvrir la discussion
Cliquez : **https://github.com/trivoallan/groove-engineering/pull/1**
Vous arrivez sur la « Pull Request » : c'est notre **salle de discussion**.
Pour aller directement là où on commente en ligne : **https://github.com/trivoallan/groove-engineering/pull/1/files**

## 3. Commenter
Deux façons :
- **Un avis général** → onglet **Conversation**, le champ tout en bas, écrire, cliquer sur le bouton **Comment**.
- **Réagir à un point précis** *(mieux)* → onglet **Files changed**, survoler la ligne qui vous intéresse, cliquer sur le petit **＋** bleu, écrire, **Add single comment**. Votre note reste épinglée à l'endroit exact.

## Trois conventions légères
- Commencer par votre **registre** : 🎼 musicologie (un fait) · 👂 ressenti (ce que vous ressentez) · ✊ politique (valeurs).
- Pointer vers le marqueur : « P3 », « Q1 »… (les points et questions sont numérotés dans le document).
- **Le désaccord est le sujet, pas un problème.** *non = malentendu.*

Pas besoin de tout lire. Allez directement aux **questions Q1–Q6** qui vous parlent. Merci de prêter votre oreille.
